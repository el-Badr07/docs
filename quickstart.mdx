Welcome to Textra! This guide will walk you through processing your first invoice using our intelligent accounting automation platform. In just a few minutes, you'll see how Textra transforms raw invoices into compliant French accounting entries.

## What is Textra?

Textra is an AI-powered accounting automation platform that processes financial documents through a sophisticated 5-phase pipeline:

1. **Data Extraction** - OCR and LLM extract structured data from documents
2. **Account Imputation** - AI maps transactions to French PCG accounts 
3. **Journal Generation** - Creates balanced double-entry journal entries
4. **Validation** - Ensures compliance and accuracy
5. **Export** - Outputs in multiple formats (FEC, CSV, XML, Sage)

## Setup and Installation

### 1. Clone and Setup Backend

<CodeGroup>
```bash Terminal
# Clone the repository
git clone https://github.com/el-Badr07/Textra.git
cd textra

# Create virtual environment
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt
```

```bash Docker
# Using Docker Compose
docker-compose up -d

# This starts:
# - MongoDB database
# - Redis cache
# - RabbitMQ message broker
# - Textra backend services
```
</CodeGroup>

### 2. Configure Environment

Create a `.env` file in the project root:

```bash .env
# Database Configuration
MONGODB_URL=mongodb://localhost:27017
MONGODB_DB_NAME=textra_accounting

# Redis Configuration  
REDIS_URL=redis://localhost:6379

# RabbitMQ Configuration
RABBITMQ_URL=amqp://localhost:5672

# LLM Provider Configuration
GROQ_API_KEY=your_groq_api_key
GEMINI_API_KEY=your_gemini_api_key

# File Storage
UPLOAD_FOLDER=./uploads
STORAGE_FOLDER=./storage

# Security
SECRET_KEY=your_secret_key_here
```

### 3. Start the Services

<Tabs>
  <Tab title="Development">
    ```bash
    # Start backend API (port 8001)
    uvicorn src.main:app --reload --host 0.0.0.0 --port 8001
    
    # Start frontend (separate terminal)
    cd frontend
    npm install
    npm run dev
    
    # Start a worker process (separate terminal)
    python -m src.message_worker.run_worker
    ```
  </Tab>
  
  <Tab title="Production">
    ```bash
    # Using Docker for production
    docker-compose -f docker-compose.yml up -d
    
    # If you prefer manual processes
    uvicorn src.main:app --host 0.0.0.0 --port 8001 &
    python -m src.message_worker.run_worker 
    ```
  </Tab>
</Tabs>

## Process Your First Invoice

### Step 1: Upload an Invoice

<AccordionGroup>
  <Accordion icon="upload" title="Web Interface">
    1. Navigate to `http://localhost:3000` 
    2. Click **"Upload Invoice"**
    3. Select a PDF or image file
    4. Wait for upload confirmation
  </Accordion>
  
  <Accordion icon="code" title="API Upload">
    ```bash
    curl -X POST "http://localhost:8001/api/accounting/invoices/upload" \
      -H "Content-Type: multipart/form-data" \
      -F "file=@/path/to/invoice.pdf" \
      -F "user_id=your_user_id"
    ```
    
    **Response:**
    ```json
    {
      "success": true,
      "data": {
        "file_id": "65a1b2c3d4e5f6789",
        "filename": "invoice.pdf",
        "upload_path": "/uploads/invoices/invoice_001.pdf"
      }
    }
    ```
  </Accordion>
</AccordionGroup>

### Step 2: Process the Invoice

Trigger the 5-phase processing pipeline:

<CodeGroup>
```bash Synchronous Processing
curl -X POST "http://localhost:8001/api/accounting/invoices/process" \
  -H "Content-Type: application/json" \
  -d '{
    "file_path": "/uploads/invoices/invoice_001.pdf",
    "user_id": "your_user_id",
    "processing_options": {
      "extract_line_items": true,
      "validate_vat": true,
      "generate_journal": true
    }
  }'
```

```bash Asynchronous Processing
curl -X POST "http://localhost:8001/api/accounting/invoices/process-async" \
  -H "Content-Type: application/json" \
  -d '{
    "file_path": "/uploads/invoices/invoice_001.pdf",
    "user_id": "your_user_id"
  }'
```
</CodeGroup>

**Processing Response:**
```json
{
  "success": true,
  "data": {
    "task_id": "task_65a1b2c3d4e5f6789",
    "status": "processing",
    "estimated_completion": "2024-01-15T10:32:00Z"
  }
}
```

### Step 3: Monitor Processing Status

Check the processing progress:

```bash
curl "http://localhost:8001/api/accounting/tasks/task_65a1b2c3d4e5f6789/status"
```

**Status Response:**
```json
{
  "success": true,
  "data": {
    "task_id": "task_65a1b2c3d4e5f6789",
    "status": "completed",
    "progress": 100,
    "current_phase": "export",
    "results": {
      "extraction_confidence": 0.95,
      "pcg_accounts_mapped": 8,
      "journal_entries_created": 3,
      "validation_passed": true
    }
  }
}
```

### Step 4: Retrieve Results

Get the processed accounting data:

<CodeGroup>
```bash JSON Export
curl "http://localhost:8001/api/accounting/tasks/task_65a1b2c3d4e5f6789/results?format=json"
```

```bash FEC Export
curl "http://localhost:8001/api/accounting/tasks/task_65a1b2c3d4e5f6789/results?format=fec" \
  --output invoice_journal.fec
```

```bash Sage Export
curl "http://localhost:8001/api/accounting/tasks/task_65a1b2c3d4e5f6789/results?format=sage" \
  --output invoice_sage.csv
```
</CodeGroup>
